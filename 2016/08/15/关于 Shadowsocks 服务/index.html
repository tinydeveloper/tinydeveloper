关于 Shadowsocks 服务

Posted by Slark on 2016-08-15

我是一名程序员，日常工作离不开科学上网，从最开始使用 bandwagonhost 一键安装 ss 服务，到后来自己购买 VPS，一步一步优化 ss，路上也踩了不少坑。不过现在可以将一个日本节点的 VPS 优化到可以流畅观看 4k 视频，还是很不错的。所以也将这些搭好的服务器共享出来，给大家提供稳定，低价的 ss 服务。

提供的节点

因为你所用到的节点都是我一个人负责维护的，包括 VPS 的购买、系统安装、优化等等，这仅仅是我的业余工作而已，我也并不指望这个能赚多少钱。我也希望大家尽量能别在找 ss 上面花时间，有个好用又不贵的 ss 就够了。

因为是自建 ss，所以节点并不会比专门提供这项服务的网站还多。不过，我会视购买的人数来确定是否需要增加节点。所有的节点信息，我会在本页面列出来。供大家参考。

下面是已开通的节点信息：

  • 日本 - 东京 节点（适合移动联通用户）
  • 美国 - 洛杉矶 节点（适合移动联通用户）
  • 台湾 节点（适合电信用户）

关于节点的稳定性

除了一些不可控的原因外，我会尽量保证节点的稳定性。当然，如果是线路抽风，偶尔速度不理想的话，我也没什么好的办法。一般情况下，连接速度还是比较稳定的。

另外，稳定性还和你的网络情况有关。比如，你使用的宽带是一些二级小运营商提供的，连接速度可能会比较差，也不稳定。有时候，骨干网络的波动也会影响到线路质量，当然这是不可控的。所以在购买前，最好先使用试用账号测试一下，觉得速度满意的话再付款。

还有一点需要说明的是，许多人认为，人多会影响速度，其实并不然。以现在的 jp-01 节点的 VPS 为例，联网带宽是没有限制的，1 CPU + 768 MB 的配置完全够跑一个没有 GUI 的 Linux 和一个占用资源本身就很小的 SS 了，而且 CPU 的计算资源主要用来做数据包的加解密。图为最近一周 VPS 的 CPU 使用情况，可以看到，在最高峰的时候，CPU 占用也仅仅只有 25% 左右，而高峰期的用户大概有 50 人左右（付费+试用）。所以内存和 CPU 并不能成为速度的瓶颈。

因此，在 VPS 端，唯一的瓶颈主要在于流量，由于我对所有用户是不限制流量的，而实际流量是有限的，我必须保证在每个计费周期内，流量必须要有 20% 的剩余(算是警戒线吧)。所以，我对于用户数量还是有控制的，如果某个 VPS 的流量不够了，那我会通过升级套餐的方式解决，保证大家的使用速度不会受到任何影响。

除去 CPU 和 内存 外（事实上，这两个指标在 Vultr 的 VPS 套餐里，也不可能成为 SS 速度的瓶颈），影响速度的原因大概有这么几种：

  • 本身网络速度不好；（这个属于客观原因，我也没办法解决）。
  • VPS 机房遇到拥堵现象；（这个属于正常现象，一般来说，白天日本节点比美国稳定，晚上反过来）。
  • 管理者人为限速；（这个我当然不会限制的）

综上所述，我会优先保证服务质量的，大家也不必担心人多了会影响速度。

账号的使用

不论是月付，还是年付，购买的 ss 账号都可以任意使用所有节点，不限速度，不限流量。如果后续有新的节点，我会通过邮件发送给你。

以下是你在使用过程中需要注意的：

  • 所有的节点都不限流量，但是不允许使用 BT 下载服务。
  • 累计注册的设备不能超过 5 台。
  • 禁止使用本服务从事违法活动（包括但不限于传播色情、赌博、暴力、恐怖以及发送垃圾邮件等违反法律法规等行为）。

为了保证服务质量，原则上不允许将账号分享给其他人，当然如果你仅仅是分享给 1~2 个朋友，我也表示认可，但是注意：累计注册的设备数量不能超过 5 台，这对于一般人来说也足够了。如果我发现某个账号有异常，我会封掉这个账号，并用邮件通知你。

邀请计划

如果你觉得我提供的服务不错，你可以邀请好友来购买我的服务，对于每成功邀请一人购买年付服务，我会返现 10 元给你。
